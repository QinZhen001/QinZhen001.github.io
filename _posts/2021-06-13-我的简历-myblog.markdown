**核心工作**：

* **自研 tme-build 构建系统**：基于 Rspack/Webpack 5 双引擎，支持多项目并行构建

**项目成果：**大仓统一管理 **80+** 个互动玩法项目，标准化研发流程，**一套代码多平台复用**，显著提升多人协作效率与产物质量，支撑玩法在多端快速迭代上线。性能优化显著，首屏加载时间 **<2s**，动画帧率稳定 **60fps**，支持多平台部署和国际化，服务日活数千万用户的直播互动场景，相比传统模式节省超 50% 开发工作量

### 统一广告 SDK

时间：2025.6 - 2025.4

**技术栈：** TypeScript + Rollup + JSBridge + Hippy + JCE 协议

**项目背景：** WeSing 和 JOOX 两款 App 的激励视频广告接入逻辑分散于各端（Web H5 / Hippy），维护成本高且上报口径不统一。设计并开发了跨 App、跨平台的统一广告 SDK，封装广告加载、展示、奖励领取、数据上报的完整生命周期，完善api文档，供各业务方开箱即用。

**核心工作：**

1. **跨平台分层架构** — 核心逻辑（状态机、上报、错误处理）抽象为平台无关层，平台层仅处理桥调用差异；Rollup 多入口构建支持 Tree-shaking，按需引入。
2. **广告播放状态机** — 覆盖客户端回调的有限状态机，双标志位机制兼容事件乱序，确保 Promise 在各边界条件下正确 resolve/reject。
3. **双轨数据上报** — 策略模式实现 JOOX / WeSing 两套上报策略，运行时按 App 标识自动切换，覆盖广告全链路节点。
4. **性能优化** — Cookie 缓存 + memoize。iOS 端播放后自动预加载提升二次填充率。
5. **错误与奖励机制** — 自定义AdError精确区分失败原因；奖励领取支持原生 / Web 双通道灵活切换。

**项目成果：**统一了 2 款 App、2 个平台的广告接入方案，覆盖 35+ 广告场景，业务接入成本从 ~3 天降至 1 天；对外 API 仅 loadAd/ showAd等核心方法，大幅降低使用门槛。
